## Added

- **PR Convergence ‚Äî Path to Merge tab** ‚Äî Dedicated convergence workspace inside every PR detail view with issue inventory, manual and auto-converge modes, and pipeline settings
- **Auto-Converge mode** ‚Äî Fully autonomous convergence loop: launches resolution rounds, polls for CI completion and comment stabilization, and optionally auto-merges when all issues are resolved
- **Issue Inventory** ‚Äî Structured tracking of all PR blockers (review comments, CI failures, bot feedback) with state management (new, sent to agent, fixed, dismissed, escalated) and source attribution (human, CodeRabbit, Copilot, Codex)
- **CTO PR convergence operator tools** ‚Äî Five new MCP tools (`getPullRequestConvergence`, `updatePullRequestConvergencePipeline`, `updatePullRequestConvergenceRuntime`, `startPullRequestConvergenceRound`, `stopPullRequestConvergence`) enabling the CTO to drive PR convergence programmatically
- **Pipeline settings** ‚Äî Per-PR configuration for auto-merge, merge method, max rounds, and rebase policy with project-wide defaults in `.ade/ade.yaml`
- **Unified model picker in convergence** ‚Äî Select any configured AI model, reasoning effort, and permission mode for each convergence round
- **ADE MCP tool namespacing** ‚Äî PR issue resolution tools exposed with `ade:` prefix in agent sessions
- **Codex MCP override mapping** ‚Äî ADE server settings correctly forwarded to Codex MCP launches
- **Reusable JSON-RPC server** ‚Äî Refactored MCP socket handling into a shared server module
- **Packed session grid** ‚Äî Resizable tile layout for the Work view with drag-handle resizing and bin-packing arrangement
- **Multi-select agent questions** ‚Äî Agent question modal supports toggling multiple options with Markdown/HTML preview
- **New Chat quick-create** ‚Äî Faster optimistic session opening with immediate tab activation
- **Turn recap UI** ‚Äî Summary row at the end of each turn showing tool invocation counts and task progress
- **Claude tool-use tracking** ‚Äî Per-invocation lifecycle tracking with start/complete events for work log status indicators
- **MCP initialize probe** ‚Äî Pre-checks MCP server availability before starting Claude sessions

## Changed

- **PR detail tabs** ‚Äî "Path to Merge" is now a permanent tab (with sparkle icon and issue count badge) alongside Overview, Files, CI/Checks, and Activity
- **Chat sessions** ‚Äî MCP tools are now wired into unified chat sessions with full model picker support across all configured providers
- **Chat provider resolution** ‚Äî Model descriptor-based provider and model selection replaces hardcoded provider switching
- **Headless MCP** ‚Äî Electron safeStorage guarded to support MCP server operation without a display
- **MCP config hardening** ‚Äî ADE query controls enforced across all MCP tool invocations
- **Chat abort handling** ‚Äî Aborts treated as interruptions that preserve chat state instead of clearing it
- **Rebase route selection** ‚Äî Active rebase needs now correctly resolved when selecting navigation routes
- **Terminal renderer** ‚Äî Simplified from WebGL/canvas/DOM to WebGL-first with DOM fallback; added fit recovery with retry on invalid dimensions
- **Work log headings** ‚Äî Human-readable labels replace generic tool identifiers; default visible entries increased to 4
- **Model catalog filtering** ‚Äî Model selector supports `available-only` mode to restrict picker to configured providers
- **Git stash actions** ‚Äî Pop, drop, and clear now refresh workspace metadata after completion
- **Composer sizing** ‚Äî New compact and grid-tile sizing modes

## Fixed

- **Convergence state guard** ‚Äî `storeConvergenceState` correctly checks for existing state before overwriting
- **MCP launch environment** ‚Äî Removed `process.env` leak from MCP launch configuration
- **MCP SDK test isolation** ‚Äî Added proper mocks for MCP SDK dependencies in test suites
- **Thinking indicator positioning** ‚Äî Fixed CSS positioning of the thinking/reasoning indicator in chat messages
- **Artifact URL handling** ‚Äî Textual artifact URLs are now ignored (only real PR branch pushes trigger updates)
- **projectRoot forwarding** ‚Äî Codex MCP launches now correctly receive the project root path
