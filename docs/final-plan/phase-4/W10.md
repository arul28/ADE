# W10: .ade/ Portable State
> Source: [OpenClaw — "file as truth" principle](https://velvetshark.com/articles/openclaw-memory-masterclass) — config files on disk are the source of truth, DB syncs from them. [Paperclip — log integrity](https://github.com/paperclipai/paperclip/blob/main/doc/SPEC.md) — append-only log hash chaining. [Symphony — dynamic config reload](https://github.com/openai/symphony/blob/main/SPEC.md) — file-watch config reload.

Dependencies: **W1** (CTO state files), **W6½** (memory engine).

W10 is live in the repo. ADE now treats `.ade/` as a canonical project-local contract with a small tracked/shareable subset and clearly named machine-local runtime buckets. Desktop startup repairs that structure automatically, MCP bootstrap resolves paths from the same shared layout helper, and the renderer exposes a dedicated project-health surface in Settings.

##### Audit Snapshot (2026-03-11)

- `apps/desktop/src/shared/adeLayout.ts` is the single source of truth for `.ade` paths used by the desktop app and MCP server bootstrap.
- `adeProjectService.ts` initializes or repairs `.ade/` on startup, writes the tracked `.ade/.gitignore`, and scrubs stale `.git/info/exclude` rules that ignored the entire directory.
- Legacy runtime folders are rehomed into canonical buckets under `artifacts/`, `transcripts/`, `cache/`, and `secrets/`.
- `logIntegrityService.ts` normalizes tracked append-only JSONL files and writes `prevHash` chain entries on new CTO/worker session log appends.
- `configReloadService.ts` watches `ade.yaml`, `local.yaml`, and `local.secret.yaml`, reloads dependent services, and emits project-state refresh events for the renderer.
- Learned skill export now writes to `.ade/skills/<slug>/SKILL.md`. `.claude/skills` remains a legacy read/index source only.

##### Git Contract

`.ade/.gitignore` is tracked and defines the machine-local subset:

- ignored files: `local.yaml`, `local.secret.yaml`, `ade.db`, `ade.db-*`, `ade.db-wal`, `embeddings.db`, `mcp.sock`
- ignored directories: `artifacts/`, `transcripts/`, `cache/`, `worktrees/`, `secrets/`

Everything else in the canonical root is intentionally shareable and visible to git. ADE no longer relies on a repo-local `.git/info/exclude` rule to hide the whole `.ade/` directory.

##### Startup Repair and Migration

`initializeOrRepairAdeProject()` runs at startup and on manual repair from Settings. It:

1. ensures the canonical tracked and ignored roots exist,
2. writes or rewrites `.ade/.gitignore` to the canonical machine-local rules,
3. removes stale `.git/info/exclude` lines for `.ade` / `.ade/`,
4. moves legacy runtime paths into their current homes:
   - `.ade/logs` → `.ade/transcripts/logs`
   - `.ade/chat-transcripts` → `.ade/transcripts/chat`
   - `.ade/chat-sessions` → `.ade/cache/chat-sessions`
   - `.ade/orchestrator` → `.ade/cache/orchestrator`
   - `.ade/packs` → `.ade/artifacts/packs`
   - `.ade/log-bundles` → `.ade/artifacts/log-bundles`
   - `.ade/github` → `.ade/secrets/github`
   - `.ade/api-keys.json` → `.ade/secrets/api-keys.json`
   - root `mission-state-*.json` and `coordinator-checkpoint-*.json` → `.ade/cache/mission-state/`
5. deletes stale clutter such as legacy `README.txt` files and stray `.DS_Store` files.

##### Validation and Health Reporting

`adeProjectService.ts` surfaces a project snapshot made of:

- **structure entries**: tracked vs ignored paths, absolute path, and presence on disk
- **cleanup status**: startup repair actions that were applied
- **config state**: shared/local/secret paths plus current trust hashes from `projectConfigService`
- **health issues**: non-blocking warnings and errors for:
  - invalid `ade.yaml` / `local.yaml` validation results surfaced by `projectConfigService`
  - invalid CTO / worker `identity.yaml` files
  - invalid CTO / worker `core-memory.json` files
  - likely secrets detected in tracked files
  - missing `.ade/local.secret.yaml` when shared config references `secretRef:`

Disk files remain the durable source of truth for CTO/worker identity state. Integrity checks trigger the same snapshot/list flows used by the CTO and worker services so file-backed state stays reconciled with runtime reads.

##### Integrity and Reload

**Append-only JSONL normalization**

- tracked integrity targets:
  - `.ade/history/missions.jsonl`
  - `.ade/cto/sessions.jsonl`
  - `.ade/agents/*/sessions.jsonl`
- `logIntegrityService.normalizeJsonlFile()` rewrites each file to the valid JSONL prefix, drops malformed trailing lines, and recomputes `prevHash` for every entry.
- new CTO and worker session log entries are appended through `logIntegrityService.appendEntry()`, which writes the correct `prevHash` chain link.

**Config reload**

- `configReloadService.ts` watches:
  - `.ade/ade.yaml`
  - `.ade/local.yaml`
  - `.ade/local.secret.yaml`
- on change it:
  - refreshes config reads,
  - reloads automation schedules from config,
  - reloads secret-backed services when `local.secret.yaml` changes,
  - emits a `config-changed` project-state event so renderer surfaces refresh in place.

##### Settings Surface

Settings now includes a dedicated **Project** section with three live views:

- **Structure**: canonical `.ade` entries, tracked vs ignored status, and startup repair summary
- **Configuration**: shared/local/secret config paths plus shared/local trust hash state
- **Health**: validation warnings, secret-scan warnings, and identity/core-memory file issues

Two actions are exposed directly in the UI:

- `Repair Structure`
- `Run Integrity Check`

The same snapshot and actions are also available through preload/IPC:

- `window.ade.project.getSnapshot()`
- `window.ade.project.initializeOrRepair()`
- `window.ade.project.runIntegrityCheck()`
- `window.ade.project.onStateEvent(...)`

##### Skill Export and Ingestion

- procedure export target: `.ade/skills/<slug>/SKILL.md`
- ingestion/index sources:
  - `.ade/skills/**/*.md`
  - `.claude/skills/**/*.md`
  - `.claude/commands/**/*.md`
  - `CLAUDE.md`
  - `agents.md`

ADE treats `.ade/skills/` as the canonical project-local skill library. Legacy `.claude/skills` content is still read and indexed, but new ADE exports do not write there.
